Map Overlay Analysis

Perform a suitability analysis in GRASS

Use map algebra to overlay maps

Data

Download nc_spf from ClassStore or http://fatra.cnr.ncsu.edu/gisfordesigners/

Download recode_hydrology.txt, recode_hydrology_areas.txt, and recode_slope.txt

Setup

start GRASS 7

in startup pannel
set GIS Data Directory path to your datasets
for example, C:\Users\myname\grassdata
for Project location select nc_spf (North Carolina, State Plane, feet)
for Accessible mapset select lake_raleigh
click Start GRASS

add the DEM 

d.rast lake_raleigh_dem@lake_raleigh

add the lake 

d.vect map=lake_raleigh@lake_raleigh fcolor=none

set region with 3 foot resolution 

g.region rast=lake_raleigh_dem@lake_raleigh res=3

add the Lake Raleigh Woods boundary 

d.vect map=lake_raleigh_woods@lake_raleigh fcolor=none

buffer greenways by 5ft 

v.buffer input=greenways_2014@lake_raleigh output=greenways distance=5

conversion

convert from vector to raster 

v.to.rast input=lake_raleigh@lake_raleigh output=lake_raleigh use=val
v.to.rast input=chancellors@lake_raleigh output=chancellors use=val
v.to.rast input=buildings_2013@lake_raleigh output=buildings use=val
v.to.rast input=hydrology_2013@lake_raleigh output=hydrology use=attr attrcolumn=FCODE labelcolumn=FTYPE
v.to.rast input=hydrology_areas_2013@lake_raleigh output=hydrology_areas use=attr attrcolumn=FCODE
v.to.rast input=floodplain_2006@lake_raleigh output=floodplain use=attr attrcolumn=SFHATF
v.to.rast input=streets_2014@lake_raleigh output=streets use=val
v.to.rast input=roads_2013@lake_raleigh output=roads use=val
v.to.rast input=trail_areas_2013@lake_raleigh output=trail_areas use=val
v.to.rast input=greenways@lake_raleigh output=greenways use=val

mask

create a mask using map algebra 

r.mapcalc "mask = if( isnull( buildings@lake_raleigh ||| chancellors@lake_raleigh ||| lake_raleigh@lake_raleigh),1,null())" --overwrite

slope

compute the slope 

r.slope.aspect --overwrite elevation=lake_raleigh_dem@lake_raleigh slope=lake_raleigh_slope format=percent

recoding

recode raster maps

on a scale from 1 to 10 where 1 is suitable and 10 is unsuitable 

r.recode --overwrite input=lake_raleigh_slope@lake_raleigh output=slope rules=recode_slope.txt
r.colors -n map=slope@lake_raleigh color=ryg
r.recode --overwrite input=hydrology_areas@lake_raleigh output=hydrology rules=C:\Users\Brendan\Documents\grassdata\recode_hydrology.txt
r.recode --overwrite input=hydrology_areas@lake_raleigh output=hydrology_areas rules=recode_hydrology_areas.txt

map overlay

overlay the maps using map algebra 

r.mapcalc "hydro = if( isnull(hydrology_areas@lake_raleigh ||| hydrology@lake_raleigh),1,10 )"
r.colors -n map=hydro@lake_raleigh color=ryg

r.mapcalc "paths = if( isnull( trails@lake_raleigh ||| greenways@lake_raleigh),10,1)"
r.colors -n map=paths@lake_raleigh color=ryg


r.mapcalc "flood = if( isnull( floodplain@lake_raleigh ),1,10)"
r.colors -n map=flood@lake_raleigh color=ryg

r.mapcalc "transit = if(isnull( streets@lake_raleigh ||| roads@lake_raleigh ),1,10)"
r.colors -n map=transit@lake_raleigh color=ryg

cost mapping

create a friction map with map overlay analysis

use r.series to overlay the hydro, paths, flood, transit, and soils maps using the average operation

n.b.: the categorized map of soils suitable for trails was downloaded from the USDA Web Soil Survey 

r.series --overwrite input=flood@lake_raleigh,hydro@lake_raleigh,paths@lake_raleigh,transit@lake_raleigh,soils_for_trails@lake_raleigh output=friction method=average
r.colors -n map=friction@lake_raleigh color=ryg

create a suitabilty map with map overlay analysis

use r.series to overlay the hydro, flood, and soils maps using the average operation 

r.series --overwrite input=hydro@lake_raleigh,floodplain@lake_raleigh,soils_for_trails@lake_raleigh output=suitability method=average
r.colors -n map=suitability@lake_raleigh color=ryg

create an accessiblity map with map overlay analysis

use r.series to overlay the paths and transit maps using the minimum operation 

r.series --overwrite input=paths@lake_raleigh,transit@lake_raleigh output=accessibility method=minimum
r.colors -n map=accessibility@lake_raleigh color=ryg

create a cost map with map overlay analysis

use r.series to overlay the accessibility and suitability maps using the average operation 

r.series --overwrite input=accessibility@lake_raleigh,suitability@lake_raleigh output=cost method=average
r.colors -n map=cost@lake_raleigh color=ryg

least cost path analysis

cumulative cost mapping with r.walk

compute the cumulative cost of walking across the terrain and the friction maps using r.walk 

r.walk -k --overwrite elevation=lake_raleigh_dem@lake_raleigh friction=friction@lake_raleigh output=walk outdir=walking_direction start_coordinates=2095852.63735,735068.392216 stop_coordinates=2093366.6134,732627.296407

compute the least cost path using r.drain 

r.drain -d --overwrite input=walk@lake_raleigh indir=walking_direction@lake_raleigh output=walking_route start_coordinates=2095852.63735,735068.392216,2093366.6134,732627.296407

cumulative cost mapping with r.cost

compute the cumulative cost allocation using r.cost 

r.cost -k --overwrite input=cost@lake_raleigh output=cost_accumulation outdir=cost_direction start_coordinates=2095852.63735,735068.392216 stop_coordinates=2093366.6134,732627.296407

compute the least cost path using r.drain 

r.drain -d --overwrite input=cost_accumulation@lake_raleigh indir=cost_direction@lake_raleigh output=cost_route start_coordinates=2095852.63735,735068.392216,2093366.6134,732627.296407
